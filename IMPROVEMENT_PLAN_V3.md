## Part I: The Fatiha Framework (Course Architecture)

### 1.1 The "Mother of the Course" Concept

Just as Al-Fatiha contains the entire Quran in miniature, create an **Opening Module** that encapsulates all core concepts:

```
THE SEVEN VERSES OF META-ANALYSIS WISDOM
════════════════════════════════════════

Verse 1: "Not every signal is truth"
         → Surrogate endpoints can deceive (CAST)

Verse 2: "Methods protect patients from our confidence"
         → Pre-registration prevents HARKing (HRT)

Verse 3: "What was hidden in plain sight?"
         → Publication bias distorts the literature

Verse 4: "The number without provenance is not a number"
         → Data integrity is non-negotiable (DECREASE)

Verse 5: "Heterogeneity is a message, not noise"
         → Explore variation, don't just suppress it

Verse 6: "Absence of evidence ≠ evidence of absence"
         → Power, silence, and what studies couldn't detect

Verse 7: "Certainty must be earned, not assumed"
         → GRADE: from evidence to recommendation
```

**Implementation**:
- [ ] Create "Opening Recitation" - 90-second video summarizing all 7 principles
- [ ] Display principles as side panel during all modules
- [ ] Daily Review pulls from these 7 principles
- [ ] Final exam tests recall and application of all 7

### 1.2 The Dialogue Structure (Two-Part Learning)

Al-Fatiha divides into God's portion (first half) and servant's portion (second half). Apply this:

| Phase | Fatiha Parallel | Course Application |
|-------|-----------------|-------------------|
| **Part 1: Receiving** | God speaks | Instructor presents concept, story, evidence |
| **Part 2: Responding** | Servant speaks | Learner applies, questions, practices |

**For Each Module**:
```
PART 1 - RECEIVING (Instructor Voice)
├── The Story (what happened)
├── The Principle (what it teaches)
├── The Method (how to apply)
└── The Warning (what goes wrong)

PART 2 - RESPONDING (Learner Voice)
├── Predict (what do you expect?)
├── Practice (try the method)
├── Reflect (what did you learn?)
└── Connect (link to other principles)
```

### 1.3 The Gateway Function

Al-Fatiha prepares the reader's mind before the Quran. Create **Pre-Module Primers**:

- [ ] Each module begins with 30-second "mindset moment"
- [ ] Question that surfaces existing beliefs/misconceptions
- [ ] Brief story hook that creates cognitive tension
- [ ] Statement of what will change by module end

## Part II: Quranic Teaching Methods

### 2.1 Storytelling (Qasas) - Evidence Reversal Narratives

The Quran uses stories of prophets to teach moral lessons. We use stories of evidence reversals to teach methodological lessons.

**Story Structure** (based on Quranic narrative patterns):
```
THE REVERSAL ARC
════════════════
1. THE SETTING: What was believed (establishment view)
2. THE SIGN: Early evidence that seemed to confirm
3. THE TEST: The rigorous trial that challenged
4. THE REVERSAL: What the evidence actually showed
5. THE LESSON: The principle extracted
6. THE WARNING: How to avoid this in future
```

**40 Stories Repository** (from Evidence_Reversal_40):
- [ ] Tag each story with which of 7 principles it teaches
- [ ] Create "Story of the Day" feature in Daily Review
- [ ] Add "hook" and "refrain" for memorability
- [ ] Include the "quiet question" for contemplation

### 2.2 Parables (Amtsal) - Extended Analogy System

Expand the analogy system with Quranic-style extended metaphors:

| Concept | Parable | Extended Teaching |
|---------|---------|-------------------|
| **Meta-analysis** | "The Council of Scholars" | Each study is a scholar with partial knowledge. The meta-analysis is the council that synthesizes their wisdom. Some speak louder (larger N), some whisper (small studies). The council's verdict (pooled estimate) weighs all voices. |
| **Heterogeneity** | "The Orchestra" | An orchestra tuning before a concert. Some variance is expected. But if the violins play in different keys (I² > 75%), something is wrong with the sheet music (the underlying hypothesis). |
| **Publication Bias** | "The Missing Witnesses" | A trial where only witnesses who saw the defendant commit the crime are allowed to testify. The jury (readers) never hears from those who saw nothing. The verdict (literature) is biased. |
| **Random Effects** | "The Traveling Merchant" | A merchant who trades in many cities. In each city, the true price varies. He can estimate the average price across cities (pooled estimate) but must account for between-city variation (tau²). |
| **Forest Plot** | "The Jury Verdict" | Each study casts a vote. The size of the vote (weight) depends on precision. The diamond is the unanimous verdict. Wide confidence intervals mean jurors are uncertain. |
| **Funnel Plot** | "The Lighthouse" | A lighthouse illuminates ships in all directions. If ships only appear on one side, something is blocking the light (publication bias) or the lighthouse is crooked (small study effects). |
| **GRADE** | "The Ladder of Trust" | You start at ground level (very low certainty). Each step up (good methodology, consistency, directness) takes you higher. Each step down (bias, imprecision) drops you back. |

**Implementation**:
- [ ] Create visual parable cards for each concept
- [ ] Add "parable mode" toggle that shows extended metaphors
- [ ] Include cultural variants for international audiences
- [ ] Test comprehension with/without parables (A/B)

### 2.4 The Khidr Method (Learning Through Paradox)

From Surat al-Kahf: Moses learns from Khidr through paradoxical actions that initially seem wrong but reveal wisdom.

**Paradox Exercises**:
```
PARADOX 1: "Adding Studies Can Widen Confidence Intervals"
Initial belief: More studies = more precision
Reveal: Adding heterogeneous studies increases tau²

PARADOX 2: "The Significant Finding That Means Nothing"
Initial belief: p < 0.05 = meaningful
Reveal: Tiny effect sizes with huge samples

PARADOX 3: "The Non-Significant Meta-Analysis That Changes Practice"
Initial belief: p > 0.05 = no effect
Reveal: Ruling out meaningful harm/benefit

PARADOX 4: "Smaller Studies Showing Bigger Effects"
Initial belief: All studies estimate the same truth
Reveal: Small study effects and publication bias
```

**Implementation**:
- [ ] "Paradox of the Week" feature
- [ ] Prediction before reveal (confidence calibration)
- [ ] Explanation of why intuition failed
- [ ] Connection to one of 7 principles

## Part III: Prophetic Pedagogy (Hadith-Based Methods)

### 3.1 Triple Repetition Pattern

The Prophet (PBUH) would repeat important points three times. Implement systematic triple exposure:

| Exposure | Format | Context |
|----------|--------|---------|
| **First (Narrative)** | Story/case study | Emotional engagement, "why this matters" |
| **Second (Technical)** | Formal explanation | Definitions, formulas, procedures |
| **Third (Application)** | Hands-on practice | Interactive exercises, real data |

**Tracking**:
- [ ] Log which concepts have received 1/2/3 exposures
- [ ] Dashboard shows "exposure completion" for each principle
- [ ] Alert if learner attempts advanced content without triple exposure to prerequisites

### 3.2 Analogy to Daily Life (River Bathing Hadith)

The Prophet explained five daily prayers with: "If there was a river at your door and you bathed in it five times daily, would any dirt remain?"

**Daily Life Analogies**:

| Concept | Daily Life Analogy |
|---------|-------------------|
| **Systematic Review** | "Like comparing ALL product reviews before buying, not just the first page" |
| **Effect Size** | "Not just 'did the diet work?' but 'how many pounds?'" |
| **Confidence Interval** | "Like GPS showing 'arrive between 5:10-5:20' vs just '5:15'" |
| **Heterogeneity** | "Reviews from New York and rural Kansas might differ systematically" |
| **Publication Bias** | "What if Amazon only showed 4-5 star reviews?" |
| **Quality Assessment** | "Checking if the reviewer actually bought and used the product" |
| **Sensitivity Analysis** | "Recalculating your budget with and without that expensive item" |

### 3.3 Visual/Graphic Teaching

The Prophet drew in the sand to explain concepts. Create visual teaching moments:

- [ ] **Hand-Drawn Style Graphics**: Deliberately sketch-like visuals for concepts
- [ ] **Step-by-Step Builds**: Forest plots that build incrementally
- [ ] **Annotation Mode**: Instructor "draws on" the data while explaining
- [ ] **Sand Drawing Aesthetic**: Optional visual style mimicking hand-drawn teaching

### 3.4 Socratic Pre-Questions

Before revealing results, always ask:

```javascript
const socraticQuestions = {
  beforeForestPlot: [
    "What direction do you expect the effect?",
    "How confident are you? (0-100%)",
    "What would change your mind?"
  ],
  beforeFunnelPlot: [
    "Do you expect symmetry?",
    "What would asymmetry suggest?",
    "Which corner might have missing studies?"
  ],
  beforeHeterogeneity: [
    "Do you expect these studies to agree?",
    "What might explain differences?",
    "Should we pool these results?"
  ]
};
```

**Implementation**:
- [ ] Mandatory prediction before every major reveal
- [ ] Track prediction accuracy over time (calibration)
- [ ] Show learners their calibration curve in dashboard

### 3.5 Situational Teaching

Teach at the point of need, not in abstract. Create "Just-in-Time" learning triggers:

- [ ] If learner's I² is high, trigger heterogeneity module
- [ ] If learner enters impossible effect size, trigger effect size explainer

- [ ] Context-aware tooltips that teach when relevant

### 8.5 Spaced Repetition Engine

**SM-2 Algorithm Implementation**:
```javascript
const spacedRepetition = {
  calculateNextReview: (concept, quality) => {
    // quality: 0-5 (0=blackout, 5=perfect recall)
    if (quality < 3) {
      // Reset to beginning
      concept.repetitions = 0;
      concept.interval = 1;
    } else {
      // Successful recall
      if (concept.repetitions === 0) {
        concept.interval = 1;
      } else if (concept.repetitions === 1) {
        concept.interval = 6;
      } else {
        concept.interval = Math.round(
          concept.interval * concept.easeFactor
        );
      }
      concept.repetitions++;
      concept.easeFactor = Math.max(1.3,
        concept.easeFactor + 0.1 - (5-quality) * 0.08
      );
    }
    concept.nextReview = addDays(today, concept.interval);
    return concept;
  }
};
```
